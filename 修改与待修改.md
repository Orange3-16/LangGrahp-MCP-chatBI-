- `SELECT * FROM goods; DROP TABLE goods` 会被拒绝。
- `DELETE FROM goods` 会被拒绝。
- `SELECT * FROM goods -- bypass` 会被拒绝。
- `SHOW TABLES` 允许执行。

### 4. 增加 schema 缓存

提交：`0fdb2c0 Cache database schema for table tool`

- `langGraph_agent/smart_data_analysis_assistant/mcp_server/statistic_db_mcp_tools.py`

已完成：

- 第一次调用或缓存过期时查询数据库 schema。


已验证：

- 第一次调用会刷新 schema 缓存。
- 第二次调用命中缓存。
- `force_refresh=True` 会强制刷新。
- LangGraph 能正常加载带 `force_refresh` 参数的 MCP 工具。

### 5. 增加 SQL 表名和字段名存在性校验

提交：`4aea759 Validate SQL schema references before execution`

修改文件：

- `langGraph_agent/smart_data_analysis_assistant/mcp_server/statistic_db_mcp_tools.py`

已完成：

- 增加 schema 元数据缓存，保存表名和字段名映射。
- 在 SQL 执行前检查 `FROM/JOIN` 引用的表是否存在。
- 检查 `alias.column` 或 `table.column` 引用的字段是否存在。
- 检查常见单表和多表 SQL 中的未限定字段是否存在于已引用表中。
- 支持 `DESCRIBE/DESC` 表名存在性校验。
- 对复杂 CTE 或子查询采用保守策略：能明确判断错误时拒绝，否则交给数据库执行错误兜底，避免误杀合法 SQL。

已验证：

- `SELECT COUNT(*) AS sports_goods_count FROM sports_goods_price_list` 允许执行并自动补 `LIMIT`。
- `SELECT product_name, product_price FROM sports_goods_price_list WHERE product_name = '跑步机'` 允许执行。
- `SELECT product_name FROM not_exist_table` 会被拒绝，提示表不存在。
- `SELECT not_exist_column FROM sports_goods_price_list` 会被拒绝，提示字段不存在。
- `SELECT s.product_name FROM sports_goods_price_list s` 允许执行。
- `SELECT s.not_exist_column FROM sports_goods_price_list s` 会被拒绝，提示字段不存在。
- `DESCRIBE sports_goods_price_list` 允许执行。
- `DESCRIBE not_exist_table` 会被拒绝，提示表不存在。

### 7. 增加 SQL 自动修复流程

提交：`cfb9c9e Add SQL auto repair flow`

修改文件：

- `langGraph_agent/smart_data_analysis_assistant/chatbi_graph/build_graph.py`
- `langGraph_agent/smart_data_analysis_assistant/chatbi_graph/my_state.py`
- `修改与待修改.md`

已完成：

- 在 LangGraph 状态中增加 `sql_repair_count`，用于限制 SQL 自动修复次数。
- 增加 SQL 修复系统提示 `SQL_REPAIR_SYSTEM_PROMPT`。
- 增加 `repair_sql_query` 节点，用于根据失败 SQL、错误信息和历史 schema 生成修复后的 `db_sql_tool` 调用。
- 增加 `should_repair_or_continue` 条件路由。
- 工具执行后如果 `db_sql_tool` 返回错误，且修复次数未超限，则进入 SQL 修复节点。
- 修复节点只绑定 `db_sql_tool`，避免修复过程调用无关工具。
- 如果模型没有按工具调用格式返回，但返回了 SQL 文本，会尝试提取 SQL 并转换为 `db_sql_tool` 调用。
- 默认最多自动修复 1 次。

可配置项：

```env
MAX_SQL_REPAIR_ATTEMPTS=1
```

流程：

```text
LLM -> db_sql_tool -> SQL错误 -> repair_sql_query -> db_sql_tool -> LLM总结
```

注意：

- SQL 修复不放在 `db_sql_tool` 内部，工具层仍只负责安全校验、schema 校验和执行。
- 修复流程在 LangGraph 层完成，便于限制次数、观察链路和后续评测。
